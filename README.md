# Sofa-CNC — платформа автоматизации чертежей каркасов мягкой мебели

Оператор загружает фото дивана → вводит 3–5 размеров → система генерирует готовые
**DXF-файлы каркаса** для ЧПУ-станка, карту раскроя и историю заказов.

```
[Фото] → [Claude Vision] → [тип + конфигурация]
                                  ↓
   [Оператор: размеры + материал + соединение]
                                  ↓
        [Параметрический движок каркасов]
                                  ↓
   [DXF-генератор] → [ZIP] + [карта раскроя] + [история]
```

## Стек
- **Backend:** Python + FastAPI
- **AI Vision:** Anthropic Claude (`claude-sonnet-4-6`)
- **DXF:** `ezdxf`
- **БД:** SQLite через SQLModel
- **Frontend:** статический HTML/JS (отдаётся самим FastAPI)

## Запуск в 3 команды

```bash
cd sofa-cnc && python3 -m venv .venv && source .venv/bin/activate
pip install -r backend/requirements.txt && cp .env.example .env   # впишите ANTHROPIC_API_KEY в .env
cd backend && uvicorn main:app --reload
```

Откройте <http://127.0.0.1:8000> — главный экран, <http://127.0.0.1:8000/orders> — история.

> Без ключа `ANTHROPIC_API_KEY` анализ фото переходит в **ручной режим**
> (оператор выбирает тип сам) — генерация DXF при этом работает полностью.

### Windows — в один двойной клик
1. Установите Python 3.10+ с <https://python.org> (при установке отметьте **«Add Python to PATH»**).
2. Двойной клик по **`run.bat`** — он создаст окружение, поставит зависимости и запустит сервер.
3. Откройте <http://127.0.0.1:8000>.

### macOS/Linux — одной командой
```bash
bash run.sh
```

## Деплой в облако (рабочая ссылка для людей)
GitHub хранит только код — живое приложение нужно запускать на хостинге с Python.
Самый простой бесплатный путь — **Render** (в репозитории есть `Dockerfile` и `render.yaml`):

1. Залейте репозиторий на GitHub.
2. На <https://render.com> → **New → Blueprint** → выберите репозиторий → Render сам прочитает `render.yaml`.
3. В настройках сервиса впишите секрет **`ANTHROPIC_API_KEY`** → нажмите **Deploy**.
4. Через пару минут получите ссылку вида `https://sofa-cnc.onrender.com` — её можно давать людям.

> ⚠️ Бесплатный тариф Render «засыпает» после простоя (первый заход ~30 сек) и имеет
> эфемерный диск — история заказов сбрасывается при передеплое. Для постоянной истории
> подключите платный диск или Postgres.
>
> ⚠️ На публичной ссылке анализ фото тратит **ваш** баланс Claude API (каждый, кто
> загрузит фото, расходует ваши деньги). Генерация DXF — бесплатна. Если ссылку дают
> широко — стоит ограничить доступ или отключить анализ фото.

## Поток работы оператора (≤ 2 минут)
1. Перетащить фото дивана → **Анализировать**.
2. Проверить/поправить тип, подлокотники, спинку (все поля редактируемые).
3. Ввести длину, глубину, высоты, выбрать материал и тип соединения.
4. **Сгенерировать чертежи** → превью деталей, таблица, карта раскроя, число листов.
5. **Скачать DXF (ZIP)** и/или **Сохранить заказ** в историю.

## Что внутри DXF
Каждая уникальная деталь — отдельный файл, масштаб 1:1, единицы — мм. Слои:
`КОНТУР`, `РАЗМЕРЫ`, `ТЕКСТ`, `ОТВЕРСТИЯ` (отверстия под выбранное соединение:
шкант Ø8/шаг150, конфирмат Ø5/шаг300, саморез Ø4.5/шаг200; шип-паз — без отверстий).

Имена файлов: `{тип}_{деталь}_{длина}x{ширина}.dxf`, например
`uglovoy_tsarga_perednyaya_1760x370.dxf`.

## Поддерживаемые типы
- **Прямой** — полный набор: царги, дно, спинка, подлокотники, ножки.
- **Угловой Г-образный** — блок A + угловой модуль + блок Б.
- **П-образный** — левый + центральный + правый блоки + 2 угловых модуля.
- **Кресло** — упрощённый прямой (длина ≤ 1000 мм).

## API
| Метод | Путь | Назначение |
|-------|------|------------|
| POST | `/api/analyze-photo` | фото → JSON с типом и конфигурацией |
| POST | `/api/generate-drawings` | параметры → детали, превью, раскрой, `download_id` |
| GET  | `/api/download/{id}` | скачать сгенерированный ZIP |
| POST | `/api/save-order` | сохранить заказ |
| GET  | `/api/orders` | список заказов |
| GET  | `/api/orders/{id}` | один заказ |
| GET  | `/api/orders/{id}/download` | повторно скачать DXF заказа |

## Структура
```
sofa-cnc/
├── backend/
│   ├── main.py                  # FastAPI: эндпоинты + раздача фронтенда
│   ├── models.py                # SQLModel (SQLite)
│   ├── vision.py                # Claude Vision
│   ├── generators/
│   │   ├── frame_calculator.py  # расчёт деталей каркаса
│   │   ├── dxf_generator.py     # генерация DXF (ezdxf) + ZIP
│   │   ├── bin_packing.py       # карта раскроя (MaxRects/FFD)
│   │   └── svg_preview.py       # SVG-превью деталей
│   └── requirements.txt
├── frontend/                    # index.html, orders.html, app.js, style.css
├── data/                        # SQLite-БД (создаётся автоматически)
├── logs/generations.log         # лог каждой генерации
├── .env.example
└── README.md
```

## Валидация
- Длина дивана 600–5000 мм (кресло 300–1000 мм).
- Глубина 300–1500 мм, высота спинки > высоты сиденья, толщина 5–60 мм.
- Размеры хранятся в мм как `float`.
